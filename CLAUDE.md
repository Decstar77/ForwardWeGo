## Project Overview

ForwardWeGo is a C++20 3D game engine with an integrated editor for creating 3D FPS Rouge like.
It uses OpenGL 3.x for rendering, ImGui for the editor UI, and a brush-based level design system.

## Build Commands

**Generate Visual Studio 2022 solution:**
```bash
premake5.exe vs2022
```
This outputs `build/atto.sln`.

**Build the project:**
Open `build/atto.sln` in Visual Studio 2022 and build the `atto` project (Debug|x64 or Release|x64).

**Run:**
```bash
.\bin\x86_64\atto_game.exe
```

No automated test suite or lint tooling — the compiler (MSVC v143, warnings-as-errors at Level 3) is the primary quality gate.

## Architecture

### Engine Singleton (`Engine`)

`atto/src/engine/atto_engine.h/cpp` — the central singleton owning all subsystems: Renderer, Input, Audio, Assets, RNG, Camera, and the scene lifecycle. Entry point is `atto_main.cpp`, which calls `engine.Run("Editor")` to start in the editor.

### Scene System

Scenes are registered via a factory pattern in `atto_scene.h`. Each scene implements `OnStart`, `OnUpdate`, `OnRender`, `OnShutdown`. The two main scenes are:

- **EditorScene** (`editor/editor_scene.h/cpp`) — map editor with 4 view modes (3 orthographic + free-cam), brush editing, entity placement, and save/load.
- **GameMapScene** (`game/game_scene_map.h/cpp`) — the playable gameplay scene.

### Map/Level System

Maps are composed of **Brushes** (axis-aligned rectangular geometry with collision) and **Entities** (Barrel, Drone_QUAD, GameMode logic, player spawns). Map files live in `assets/maps/` and are serialized via `editor/editor_property_serializer.h/cpp`. Map logic (raycasting, entity management) is in `game/game_map.h/cpp`.

### Renderer

`atto/src/engine/atto_renderer.h/cpp` — OpenGL renderer handling model rendering (lit/unlit shaders), animated models, debug lines, and skybox. Model caching is in `renderer/atto_render_model.h/cpp`.

### Key Namespaces/Types

All engine code is under the `atto` namespace. Core types (fixed-size integers, assertions, platform detection) are defined in `atto_core.h`. Math uses GLM; fixed-point math uses FPM (24.8 format).

## Project Structure

```
atto/src/
├── engine/      # Core: engine, renderer, input, audio, assets, math
├── game/        # Gameplay: map, entities, player controller
├── editor/      # Editor scene and property serialization
└── win32/       # Windows platform layer
assets/
├── maps/        # .map files (text serialization)
└── ...          # Models (FBX/GLB), fonts, skybox
build/           # Generated by CMake (not committed)
bin/             # Build output
```

## Compiler Settings

- C++20 (`/std:c++20`), exceptions off (`/EHsc-`), RTTI off (`/GR-`)
- Multi-processor compilation enabled
- Linked: `opengl32`, `openal32`, `assimp-vc143-mt`, `Dbghelp`, `ws2_32`, `winmm`
- Warnings suppressed: C4100 (unreferenced param), C4201 (nameless struct), C4702 (unreachable code)
